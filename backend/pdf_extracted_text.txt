Temple Automation Software – 
DEVELOPMENT BRD (Version 1.0) 
 
Prepared for: Development Team 
Project Type: SaaS Temple Management Platform 
Modules: Lite / Pro / Pro Max 
Key Goal: Automate all temple operations including Poojas, Donations, Accounts & Devotee 
Management 
1. System Overview 
Temple Automation Software is a cloud-based platform designed to manage temple 
operations digitally. The system supports online bookings, donation tracking, daily pooja 
printing, festival management, staff handling, and automated reporting. 
The product will be delivered in three tiers: Lite, Pro, and Pro Max. 
2. Core Features & Deliverables 
Below is the development breakdown of each module. 
3. Module 1: Devotee Management 
Functional Requirements: 
• Ability to create, edit, update devotee profiles. 
• Store name, phone, email, gothra, nakshatra, address, ID proof. 
• Booking history and donation history. 
• Search filter by name, phone, date, pooja type. 
• Export list (PDF/Excel). 
UI Requirements: 
• Table with pagination, filters, edit/view action. 
• Add/Edit form with validations. 
Developer Notes: 
• Store MasterData (Gothra, Nakshatra) in a lookup table. 
• Use unique phone number for identification.

4. Module 2: Pooja & Ritual Management (MAIN MODULE) 
Functional Requirements: 
• CRUD for pooja types. 
• Set pooja timings (multiple in a day). 
• Assign priests to poojas. 
• Track capacity (optional). 
• Auto fetch online bookings & offline entries. 
• Generate daily pooja list for printing. 
• Festival-specific pooja creation. 
Special Requirement: DAILY POOJA PRINT SYSTEM: 
Functional Requirements: 
• Pushpanjali 
• Archana 
• Abhishekam 
• Usha Pooja 
• Homam 
• Special Festival Poojas 
Print layout should include 
• Devotee Name 
• Gotra 
• Nakshatra 
• Contact number 
• Pooja name 
• Time slot 
• Amount/Offering 
• Notes to priest 
Technical Requirements 
• Auto-merge online + offline pooja entries 
• Auto-sort by pooja time 
• API endpoint: 
/pooja/daily-print?date=DD-MM-YYYY
